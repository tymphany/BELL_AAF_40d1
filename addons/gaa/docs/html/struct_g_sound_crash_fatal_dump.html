Detailed Description

Data structure defining the required data when a crash dump occurs

Field Documentation

◆ custom

GSoundCrashDumpElement custom

Custom information for Target.

◆ panic_file

GSoundCrashDumpElement panic_file

Contains the file, function, and line number that caused the current crash. This information should be presented as a formatted NULL terminated ASCII string.

The length of this field should be limited to 64 bytes

◆ reason

GSoundResetReason reason

Contains the source or reason for the current crash dump. If device did not crash, this field should be populated with GSOUND_RESET_TYPE_NORMAL

◆ registers

GSoundCrashDumpElement registers

A binary array of core MCU registers captured at the time of the crash (assert, exception, hardfault).

The length of this field should be limited to 80 bytes

◆ stack

GSoundCrashDumpElement stack

A binary array of the current stack location captured at the time of the crash (assert, exception, hardfault).

The length of this field should be limited to 384 bytes
